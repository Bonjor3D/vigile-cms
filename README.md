# Vigile CMS

Визуальный CMS-редактор для создания веб-страниц через drag-and-drop, с Node Editor для глобальных CSS/JS скриптов.

---

## Требования

| Компонент | Версия |
|-----------|--------|
| [Node.js](https://nodejs.org/) | v22+ (LTS) |
| [.NET SDK](https://dotnet.microsoft.com/download) | 10.0+ |

База данных SQLite создаётся автоматически при первом запуске — ничего настраивать не нужно.

---

## Быстрый старт

```bat
setup.bat   — установит зависимости (npm install + dotnet restore)
start.bat   — запустит backend и frontend
```

## Вручную

```bash
# Backend (1-й терминал)
cd backend
dotnet run

# Frontend (2-й терминал)
cd frontend
npm install
npm run dev
```

- **Frontend:** http://localhost:5173
- **Backend:**  http://localhost:5000

## Как пользоваться

### 1. Вход в систему

Откройте http://localhost:5173/admin/login.  
Первый вход автоматически создаёт аккаунт — достаточно ввести логин и пароль.

### 2. Менеджер сайта (`/admin/manager`)

Здесь отображаются все созданные страницы, а также глобальные шапка и подвал.  
Кнопка **"New Page"** создаёт новую страницу с заголовком, slug и endpoint.

### 3. Редактор страниц

Нажмите **"Edit"** на любой странице в менеджере.

**Режимы:**
- **Edit** (кнопка в правом верхнем углу) — включает визуальное редактирование
- **View** — предпросмотр страницы как она будет выглядеть на сайте, для перехода в этот режим, нужно выйти из режима редактирования (не забудьте сохранить), по той же кнопке в углу

**Работа с элементами:**
- **Левая панель → вкладка Add** — перетащите элемент на страницу
- **Клик по элементу** — выделение (синяя рамка)
- **Правая панель (DOM Tree)** — иерархия элементов, можно перетаскивать для изменения порядка
- **Левая панель → Style** — CSS-свойства выделенного элемента (размеры, отступы, типографика, фон, края, тени, трансформации, анимации и т.д.)
- **Адаптивность:** переключайте Desktop / Tablet / Mobile в верхней панели — стили для каждого брейкпоинта задаются отдельно (т.е. при выборе режима, изменения сохраняются исключительно под него)

**Горячие клавиши (в Edit Mode):**

| Клавиша | Действие |
|---------|----------|
| Ctrl+S | Сохранить |
| Ctrl+Z | Отмена |
| Ctrl+Shift+Z | Повтор |
| Delete | Удалить элемент |
| Ctrl+D | Дублировать элемент |

### 4. Глобальные скрипты (Node Editor) **В СТАДИИ РАЗРАБОТКИ, РАБОТАЮТ ОЧЕНЬ ПЛОХО И ТРЕБУЮТ ОЧЕНЬ БОЛЬШИХ ДОРАБОТОК**

Кнопка **"Scripts"** в верхней панели редактора.

Визуальный node-based редактор для генерации **глобального CSS и JS**, который подключается на все страницы сайта.

- **Файлы** — вкладки вверху, можно создавать несколько CSS- и JS-файлов
- **Палитра узлов (слева)** — перетащите узел на рабочее поле
  - CSS: селекторы (класс, элемент, псевдо, медиа-запрос), свойства (цвет, размер, фон,边框, тень, flex/grid, анимация и т.д.)
  - JS: события (click, load, scroll, таймер), действия (fetch API, console, redirect, raw JS), логика (условия, переменные, математика)
- **Соединения** — перетащите с выходного порта (кружок справа) на входной (кружок слева)
- **Правый клик** по узлу — контекстное меню (Delete, Duplicate, Select Connected)
- **Правый клик + drag** по пустому полю — красная линия разреза; пересечённые соединения разбиваются с точкой-ретранслятором
- **Панель Code (справа)** — скомпилированный CSS/JS в реальном времени
- **Save** — компилирует все файлы и сохраняет в глобальные настройки сайта

### 5. Тесты (квизы)

- Создание: `/admin/manager` → раздел Tests → **New Test**
- Редактирование: добавляйте вопросы (single/multiple choice), настраивайте scoring (all or nothing / proportional / subtractive)
- Прохождение: публичная ссылка `/test/:id` — сбор группы, ФИО, таймер, подсчёт переключений вкладок (счет переключений вкладок, проверка на момент жульничества. Т.е. если переключений было много, то скорее всего пользователь жульничал и параллельно гуглил вопросы)
- Результаты: `/test/:id/results` — таблица попыток, детализация ответов каждого пользователя

### 6. Файловый менеджер

Кнопка **"Files"** в редакторе — загрузка, создание, удаление, переименование, копирование, перемещение файлов в директории `Storage/` на beckend.

## API (основные endpoints)

Все API находятся под префиксом `/api`.

### Auth

| Метод | Путь | Описание |
|-------|------|----------|
| POST | `/api/auth/login` | Вход (аккаунт создаётся автоматически) |
| GET | `/api/auth/me` | Проверка сессии |

### Pages

| Метод | Путь | Описание |
|-------|------|----------|
| GET | `/api/pages` | Список страниц |
| GET | `/api/pages/:slug` | Получить страницу |
| GET | `/api/pages/by-endpoint/:endpoint` | Получить по endpoint (используется на публичном сайте) |
| POST | `/api/pages` | Создать страницу |
| PUT | `/api/pages/:id` | Обновить страницу |
| DELETE | `/api/pages/:id` | Удалить страницу |

### Tests

| Метод | Путь | Описание |
|-------|------|----------|
| GET | `/api/tests` | Список тестов |
| POST | `/api/tests` | Создать тест |
| POST | `/api/tests/:id/submit` | Отправить ответы |
| GET | `/api/tests/:id/attempts` | Попытки теста |

### Settings

| Метод | Путь | Описание |
|-------|------|----------|
| GET | `/api/settings` | Настройки сайта (name, globalCss, globalJs, header, footer, themeVariables) |
| PUT | `/api/settings` | Обновить настройки |

### Assets / Files

| Метод | Путь | Описание |
|-------|------|----------|
| GET | `/api/assets` | Список загруженных файлов |
| POST | `/api/assets/upload` | Загрузить файл |
| GET | `/api/files?path=` | Содержимое директории Storage/ |
| POST | `/api/files/upload?path=` | Загрузить в Storage/ |

## Возможности платформы

- **Визуальный drag-and-drop редактор** — 30+ HTML-элементов (div, section, heading, paragraph, image, video, button, form, flex/grid контейнеры и т.д.)
- **Полный CSS-контроль** — размеры, отступы, типографика, фон, бордеры, тени, трансформации, фильтры, анимации
- **Адаптивность** — отдельные стили для Desktop, Tablet, Mobile
- **Node Editor** — визуальное программирование глобальных CSS/JS скриптов через графы узлов
- **Многостраничность** — создание и управление множеством страниц с иерархией
- **Шапка / Подвал** — глобальные для всех страниц
- **Переиспользуемые элементы** — создавайте элементы один раз, редактируйте централизованно
- **Шаблоны** — сохраните любой div как шаблон и вставляйте в другие страницы
- **Тёмная тема** — переключение light/dark с кастомизируемыми CSS-переменными
- **Система тестирования** — создание квизов, прохождение с таймером, подсчёт результатов
- **Файловый менеджер** — загрузка и управление файлами через браузер
- **История изменений** — Undo/Redo (до 50 шагов), ревизии страниц на сервере
